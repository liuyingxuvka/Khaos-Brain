# Automatic Dream contract

Dream is the immutable LogicGuard model-verification lane. Sleep is the only owner that turns evidence into canonical model or mesh revisions, readable card projections, candidates, confidence, lifecycle status, or central predictive history.

Run `python .agents/skills/local-kb-retrieve/scripts/kb_dream.py --json` from the repository root.

Required behavior:

- Build a stable decision-relevant fingerprint from canonical route, hypothesis, source ids and content digests, and prior applicable outcome. Exclude volatile run, time, model, prompt, and thread values.
- Pin the exact authority generation, model revision, root node and ArgumentBlock, and ModelMesh revision. Missing exact authority is a visible failure; never fall back to YAML or a floating head.
- Load compact prior closures before broad experiment work.
- Reuse `no_delta_closed` for unchanged evidence without a new experiment, central history write, candidate, observation, or handoff.
- Select only a bounded route-deduplicated set that clears the value and executability gates. A no-op is success.
- Plan a bounded suite for evidence-removal, assumption-removal, rebuttal-strengthening/counterexample, boundary-pressure, cross-edge-removal, and neighbor-pin-replacement; execute every applicable path separately against the pinned immutable model.
- Record design, safety tier, allowed writes, rollback, validation plan, and explicit result criteria before execution.
- Keep experiment writes under the current Dream run root.
- Emit one typed idempotent Sleep handoff for a material model gap. Include the exact generation/model/mesh identities, affected node or edge ids, gap kind, evidence fingerprint, result digest, provenance, and requested disposition.
- Prove the canonical generation and pinned revisions are unchanged before closure.
- Never directly modify cards, models, meshes, projections, candidates, confidence, taxonomy, lifecycle state, predictive observations, or central KB history.
- Do not ask a human to read result files or make routine selections. Stop only at a real external-action or safety boundary.

Report the run id, pinned LogicGuard identities, input digest, evaluated fingerprints, tested perturbations, evidence deltas, model gaps, suppression/no-delta counts, selected experiments, canonical-generation-unchanged proof, validation classifications, handoff ids, blockers, and final state.
